4. **具身智能（Embodied AI）**：Agent 控制物理机器人。
5. **Agent 即服务（AaaS）**：Agent 产品化和商业化。
6. **上下文工程**：从 Prompt Engineering 到 Context Engineering。

## 二、高频面试题

### Q1：2025 年 AI Agent 领域最重要的技术突破有哪些？（⭐⭐⭐⭐⭐）

**参考答案**：

1. **协议标准化三件套**：
   - MCP（Agent ↔ 工具）：Anthropic 推出，成为事实标准。
   - A2A（Agent ↔ Agent）：Google 推出，解决多 Agent 互操作。
   - AG-UI（Agent ↔ 前端）：Agent 与用户界面的交互标准。

2. **通用 Agent 产品爆发**：
   - Devin（自主编程）、OpenAI Operator（浏览器操控）、Claude Code（终端编程）。
   - 这些产品验证了 Agent 从 Demo 到商业的可行性。

3. **推理模型（Reasoning Model）崛起**：
   - o1、o3、DeepSeek-R1 等通过强化学习+长思考链显著提升推理能力。
   - 与标准 LLM 的区别：花更多推理时间换取更高准确率。

4. **上下文工程**成为共识：不再只优化 Prompt 措辞，而是系统性管理整个上下文。

### Q2：什么是多模态 Agent？它比纯文本 Agent 多了哪些能力？（⭐⭐⭐⭐⭐）

**参考答案**：

多模态 Agent 能同时处理文本、图像、音频、视频等多种数据模态。

**新增能力**：
1. **视觉理解**：直接"看"屏幕截图，理解 UI 界面，执行 GUI 自动化测试。
2. **语音交互**：直接听取和回复人类语音指令。
3. **视频分析**：处理摄像头实时视频流。
4. **跨模态推理**：结合图表、代码截图、文字说明做综合分析。

**典型应用**：
- 浏览器自动化（如 OpenAI Operator 看屏幕操作网页）。
- 视觉质检（工业产线缺陷检测）。
- 多模态文档理解（包含图表的 PDF 分析）。

### Q3：什么是推理模型（如 o1/o3/DeepSeek-R1）？和标准 LLM 有何区别？（⭐⭐⭐⭐⭐）

**参考答案**：

推理模型在回答前会进行**较长时间的内部思考**（Chain-of-Thought），通过强化学习训练消耗更多推理时间来换取更高的推理准确率。

| 维度 | 标准 LLM | 推理模型 |
|------|---------|---------|
| 思考方式 | 快速直觉回答 | 深度思考后回答 |
| 训练方式 | SFT + RLHF | SFT + RL（含推理奖励） |
| 推理延迟 | 低 | 高（但质量更好） |
| 擅长任务 | 创意、通用对话 | 数学、编程、逻辑推理 |
| Token 消耗 | 低 | 高（思考过程消耗额外 Token） |

### Q4：具身智能（Embodied AI）是什么？和 LLM Agent 有什么关系？（⭐⭐⭐）

**参考答案**：

具身智能指 AI 系统拥有"身体"——能够在物理世界中感知和行动的 Agent。

**与 LLM Agent 的关联**：
1. LLM 作为"大脑"进行高层次规划和自然语言理解。
2. 视觉模型处理摄像头输入。
3. 运动控制模型将指令转化为物理动作。
4. 记忆系统存储环境地图和操作经验。

**挑战**：物理安全性、实时性要求、环境不确定性。

### Q5：Agent 的商业化路径有哪些？（⭐⭐⭐⭐）

**参考答案**：

2025 年的主要商业化方向：
1. **垂直领域 Agent SaaS**：客服、法律、金融等特定行业的 Agent 服务。
2. **开发者工具**：AI 编程助手（Cursor、Windsurf、Claude Code）。
3. **企业自动化**：RPA + Agent 的智能流程自动化。
4. **Agent 基础设施**：推理引擎（vLLM）、编排框架（LangGraph）、可观测性（LangSmith）。
5. **Agent 即服务（AaaS）**：将训练好的 Agent 作为 API 服务提供。

### Q6：未来 Agent 面试会更关注哪些方向？（⭐⭐⭐⭐）

**参考答案**：

基于 2025 年趋势判断：
1. **协议标准化**：MCP/A2A 熟悉度成为加分项。
2. **架构思维**：不只写对 Prompt，要能设计生产级 Agent 系统。
3. **安全与合规**：Guardrails、权限控制、审计成为必考。
4. **多模态能力**：视觉/语音 Agent 开发经验。
5. **成本优化**：模型路由、缓存策略等工程优化。
6. **实战项目**：有完整的、可量化的 Agent 项目经验（最好是开源的）。
