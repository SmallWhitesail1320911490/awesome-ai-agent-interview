# 第 8 章：工程实践与系统设计

> 真正的挑战在于如何将 Agent 从 Demo 推向生产。本章聚焦工程落地经验。
> 内容来源：小林coding、牛客网、CSDN、YouTube 等。

## 一、核心知识点总结

### 1.1 Prompt Engineering 最佳实践
- 角色设定 → 任务描述 → 输出格式 → 约束条件 → 示例
- 分隔符区分不同信息区域
- 使用结构化输出（JSON Mode）

### 1.2 生产环境关键挑战
- 成本控制、延迟优化、安全防护
- 可观测性和监控
- 错误处理和降级策略

## 二、高频面试题

### Q1：如何系统性地进行 Prompt Engineering？（⭐⭐⭐⭐⭐）

**参考答案**：

**结构化 Prompt 模板**：
```
[System]: 角色设定 + 全局约束
[Context]: 背景信息 / 检索结果
[Task]: 具体任务描述
[Format]: 期望的输出格式
[Examples]: Few-shot 示例（可选）
[Constraints]: 限制条件（不要做什么）
```

**核心技巧**：
1. **具体而非模糊**："列出3个要点，每个50字以内" > "简要回答"。
2. **分隔符隔离信息**：用 `---` 或 `<context>` 区分不同区域。
3. **角色扮演**：设定专家身份提升输出质量。
4. **负面约束**：明确说"不要编造"比只说"准确回答"更有效。
5. **迭代优化**：建立A/B测试框架，对比不同 Prompt 的效果。

### Q2：如何控制 Agent 系统的成本？（⭐⭐⭐⭐⭐）

**参考答案**：

1. **模型路由（Model Routing）**：简单任务用小模型（GPT-4o-mini），复杂任务才用大模型（GPT-4o）。
2. **缓存策略**：
   - Semantic Cache：相似 Query 的结果直接复用。
   - KV Cache：减少重复计算。
3. **Prompt 精简**：减少不必要的上下文信息和冗余示例。
4. **批量处理**：非实时任务批量 API 调用（通常有折扣）。
5. **Token 预算**：为每个 Agent 设置 Token 上限。
6. **监控告警**：实时监控 Token 消耗，异常时触发告警。
7. **早停机制**：Agent 已获得足够信息时立即终止循环。

### Q3：如何优化 Agent 的响应延迟？（⭐⭐⭐⭐⭐）

**参考答案**：

1. **并行化**：独立的工具调用并行执行。
2. **流式输出（Streaming）**：边生成边返回结果。
3. **预取策略**：预判可能需要的信息提前检索。
4. **模型路由**：优先用小模型尝试，失败再升级大模型。
5. **缓存**：高频查询缓存结果。
6. **推理优化**：使用 vLLM 等高性能推理引擎。
7. **异步架构**：非阻塞的事件驱动设计。

### Q4：生产环境中 Agent 的可观测性如何设计？（⭐⭐⭐⭐⭐）

**参考答案**：

1. **Trace（链路追踪）**：记录每一步推理和工具调用的完整链路。
2. **Metrics（指标监控）**：
   - 延迟（P50/P95/P99）
   - Token 消耗
   - 成功率
   - 工具调用次数
3. **Logging（日志）**：每步的输入、输出、决策原因。
4. **告警规则**：异常指标触发自动告警。
5. **可视化面板（Dashboard）**：LangSmith、LangFuse 等工具。
6. **回放/调试**：支持完整 Session 的回放和单步调试。

### Q5：如何处理 Agent 系统中的失败和异常？（⭐⭐⭐⭐）

**参考答案**：

**分层处理策略**：
1. **工具层**：超时控制 + 指数退避重试 + 幂等性保障。
2. **Agent 层**：
   - 最大步骤数限制（防止无限循环）。
   - 错误信息反馈给 LLM，让它自行调整策略。
   - Fallback 到备用工具或直接用 LLM 知识回答。
3. **系统层**：
   - 熔断机制（Circuit Breaker）。
   - 降级策略（返回缓存结果或默认回答）。
   - 状态持久化（支持任务中断后恢复）。
4. **人工兜底**：异常无法自动恢复时升级至人工处理。

### Q6：SDK（如 LangChain）和手写 Agent 循环的取舍？（⭐⭐⭐⭐）

**参考答案**：

| 维度 | SDK (LangChain等) | 手写 Agent 循环 |
|------|-------------------|-----------------|
| 开发速度 | 快，开箱即用 | 慢，需自行实现 |
| 灵活性 | 受限于框架设计 | 完全可控 |
| 调试难度 | 高，抽象层多 | 低，逻辑透明 |
| 维护成本 | 依赖框架升级 | 自主维护 |
| 适用阶段 | 原型验证、MVP | 生产级系统 |

**建议**：
- 早期探索和 POC 用 LangChain 快速验证。
- 生产系统倾向于手写核心循环或使用轻量框架（如 LangGraph、Pydantic AI）。
- 避免在生产中过度依赖高层抽象。

### Q7：如何设计模型选择策略（Model Routing）？（⭐⭐⭐⭐）

**参考答案**：

1. **基于任务复杂度**：
   - 简单任务（分类、实体提取）→ 小模型（GPT-4o-mini、Claude Haiku）。
   - 复杂任务（多步推理、代码生成）→ 大模型（GPT-4o、Claude Sonnet）。
2. **基于置信度升级**：先用小模型，置信度低则升级大模型。
3. **基于成本预算**：设定每次请求的成本上限。
4. **A/B 测试**：对比不同模型在特定任务上的效果和成本。

### Q8：如何保障 Agent 系统的安全性？（⭐⭐⭐⭐⭐）

**参考答案**：

1. **Prompt 注入防护**：输入过滤、指令隔离、系统提示保护。
2. **输出安全审查**：检测和过滤有害、敏感内容。
3. **工具权限最小化**：Agent 只能调用明确授权的工具和操作。
4. **敏感操作确认**：涉及支付、删除、发送等操作需人工确认。
5. **沙箱执行**：代码类工具在隔离环境中运行。
6. **审计日志**：完整记录所有操作供事后审查。
7. **Guardrails 框架**：使用 NeMo Guardrails 等可配置的安全护栏。
