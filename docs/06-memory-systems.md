# 第 6 章：记忆系统设计

> Agent 的记忆能力决定了它能否跨越单次对话的限制，实现真正的持续学习和个性化服务。
> 内容来源：牛客网、CSDN、小林coding、Medium 等。

---

## 一、核心知识点总结

### 1.1 记忆类型

| 类型 | 说明 | 生命周期 | 典型实现 |
|------|------|----------|----------|
| **感觉记忆** | 原始输入数据的瞬时缓存 | 毫秒级 | Token Embedding |
| **短期/工作记忆** | 当前会话的上下文 | 会话级 | Context Window |
| **长期记忆** | 跨会话持久化的知识 | 永久 | 向量数据库 / 关系数据库 |
| **情景记忆** | 具体事件和经历 | 永久 | 向量检索 |
| **语义记忆** | 结构化知识和概念 | 永久 | 知识图谱 |
| **程序性记忆** | 技能和操作模式 | 永久 | 模型权重（微调） |

### 1.2 上下文工程（Context Engineering）

上下文工程是 2025 年的新热词，是 Prompt Engineering 的升级版。核心关注点：
- **什么信息该放入上下文**：精选最相关的记忆和检索结果。
- **信息如何组织**：结构化排列，避免"Lost in the middle"问题。
- **上下文压缩**：在不丢失关键信息的前提下减小 Token 占用。
- **动态上下文管理**：随对话推进，智能替换不再相关的上下文。

## 二、高频面试题

### Q1：Agent 中短期记忆和长期记忆分别如何实现？（⭐⭐⭐⭐⭐）

**参考答案**：

**短期记忆（Working Memory）**：
- 本质是当前会话的消息历史，直接存放在 LLM 的 Context Window 中。
- 管理机制：
  - **滑动窗口**：只保留最近 N 轮对话。
  - **摘要压缩**：将历史对话用 LLM 摘要后替换原文。
  - **Token 预算**：动态控制每部分上下文的 Token 分配。

**长期记忆（Long-term Memory）**：
- 跨越会话持久存储的信息。
- 实现方式：
  - 将对话关键信息 Embedding 后存入**向量数据库**。
  - 结构化信息存入**关系型数据库或图数据库**。
  - 使用 RAG 在需要时召回相关历史。
  - 用户偏好等可用 KV 存储。

### Q2：上下文窗口管理有哪些常见策略？（⭐⭐⭐⭐⭐）

**参考答案**：

1. **滑动窗口**：保留最近 N 条消息，丢弃更早的。
2. **摘要压缩**：用 LLM 将历史对话摘要为简短描述。
3. **分层管理**：
   - 最新消息保留原文。
   - 较早消息保留摘要。
   - 更早的存入长期记忆，按需检索。
4. **重要性评分**：给每条消息打权重分，优先保留高权重内容。
5. **Token 预算分配**：将 Context Window 划分为固定区域（系统提示、工具定义、对话历史、检索结果等）。

### Q3：什么是上下文工程（Context Engineering）？它和 Prompt Engineering 有什么区别？（⭐⭐⭐⭐⭐）

**参考答案**：

**Prompt Engineering**：关注"怎么写指令"——优化 System Prompt 和 User Prompt 的措辞。

**Context Engineering**：关注"给模型看什么信息"——是更宏观的工程问题：
1. **信息选择**：从记忆、检索、工具结果中筛选最相关的信息。
2. **信息组织**：决定各部分信息的排列顺序和格式。
3. **信息压缩**：在保持关键信息的前提下减少 Token 使用。
4. **动态管理**：随任务进展实时调整上下文内容。

**2025 年共识**：在 Agent 系统中，Context Engineering 的重要性已超过 Prompt Engineering。

### Q4：什么是 "Lost in the Middle" 问题？如何解决？（⭐⭐⭐⭐）

**参考答案**：

**问题**：研究发现 LLM 对上下文中间位置的信息关注度最低，容易忽略中间内容。模型更倾向于关注开头和结尾的信息。

**解决方案**：
1. **重要信息前置或后置**：将最关键的检索结果放在上下文的开头或末尾。
2. **重排序**：使用 Reranker 将最相关内容排到最前面。
3. **分段处理**：将长上下文分段先摘要再合并。
4. **使用支持长上下文的模型**：如 Claude 3.5（200K）、Gemini 1.5（1M）。

### Q5：Agent 什么时候该存储记忆，什么时候不该？（⭐⭐⭐⭐）

**参考答案**：

**应存储**：
- 用户明确表达的偏好和个人信息。
- 任务执行的关键结论和决策。
- 被验证正确的中间结果。
- 用户反馈和纠错信息。

**不应存储**：
- 时效性极短的信息（如天气、股价）。
- 敏感的个人隐私数据（需符合隐私政策）。
- 临时的中间推理过程。
- 大量冗余的原始数据。

**设计原则**：数据敏感性、存储空间成本、检索效率三者的平衡。

### Q6：如何设计一个高效的记忆检索机制？（⭐⭐⭐⭐）

**参考答案**：

1. **多维索引**：时间维度 + 语义维度 + 实体维度。
2. **混合检索**：向量相似度（语义）+ 关键词匹配（精确）+ 时间衰减因子。
3. **分层检索**：先粗召回大范围、再精排筛选。
4. **记忆重要性评分**：基于访问频率、相关性和新鲜度的综合评分。
5. **记忆合并**：相似的记忆自动合并为更完整的条目。

### Q7：Generative Agents（斯坦福小镇）的记忆系统如何设计？（⭐⭐⭐）

**参考答案**：

斯坦福 Generative Agents 论文中的记忆系统：
1. **记忆流（Memory Stream）**：所有经历按时间序列存储。
2. **检索机制**：基于三个维度的加权评分：
   - **时效性（Recency）**：最近的记忆权重更高。
   - **重要性（Importance）**：用 LLM 给每条记忆评分 1-10。
   - **相关性（Relevance）**：与当前查询的语义相似度。
3. **反思（Reflection）**：定期让 Agent 对近期记忆进行高层次总结和推理。

这是 Agent 记忆系统设计的经典参考。
