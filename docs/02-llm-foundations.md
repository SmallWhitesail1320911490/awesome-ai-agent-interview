# 第 2 章：LLM 基础知识

> 本章是 Agent 面试的基础前提——你必须深入理解大语言模型的底层原理。
> 内容来源：DataCamp、GeeksforGeeks、各大中文面经社区等。

---

## 一、核心知识点总结

### 1.1 Transformer 架构
- Encoder-Decoder 结构（原始）、Decoder-only（GPT 系列）、Encoder-only（BERT）
- Self-Attention 机制：Q、K、V 的计算方式
- Multi-Head Attention：并行多组注意力头捕获不同维度特征
- 位置编码（Positional Encoding）：RoPE、ALiBi 等

### 1.2 Tokenization
- BPE（Byte-Pair Encoding）
- SentencePiece / WordPiece
- Token 与字符/词的关系

### 1.3 微调技术
- **SFT（Supervised Fine-Tuning）**：用标注数据微调
- **RLHF**：基于人类反馈的强化学习
- **DPO（Direct Preference Optimization）**：直接偏好优化，RLHF 的简化替代
- **PEFT/LoRA**：参数高效微调，冻结大部分参数只训练低秩矩阵
- **QLoRA**：量化后的 LoRA

### 1.4 推理参数
- **Temperature**：控制生成随机性。0 → 确定性，1+ → 更随机。
- **Top-p / Top-k**：采样策略。
- **解码策略**：Greedy、Beam Search、Nucleus Sampling。

## 二、高频面试题

### Q1：简述 Transformer 的核心结构及其相对于 RNN/LSTM 的优势。（⭐⭐⭐⭐⭐）

**参考答案**：

Transformer 核心由 Self-Attention 和 Feed-Forward Network 组成：
1. **Self-Attention**：对输入序列中的每个 Token，计算它与所有其他 Token 的关联权重，公式为 $\text{Attention}(Q,K,V) = \text{softmax}(\frac{QK^T}{\sqrt{d_k}})V$。
2. **Multi-Head Attention**：并行运行多个 Attention Head，捕获不同子空间的特征。
3. **Position-wise FFN**：对每个位置独立做非线性变换。

**相对于 RNN/LSTM 的优势**：
- **并行计算**：摆脱了序列的逐步依赖，训练速度大幅提升。
- **长距离依赖**：Attention 直接连接任意两个位置，不受步长限制。
- **灵活性**：适配各种下游任务（生成、分类、翻译等）。

### Q2：Encoder-only、Decoder-only、Encoder-Decoder 三种架构的区别和代表模型？（⭐⭐⭐⭐）

**参考答案**：

| 架构 | 特点 | 代表模型 | 适用任务 |
|------|------|----------|----------|
| Encoder-only | 只有编码器，双向注意力 | BERT、RoBERTa | 分类、NER、句子相似度 |
| Decoder-only | 只有解码器，单向（因果）注意力 | GPT 系列、LLaMA、Mistral | 文本生成、对话 |
| Encoder-Decoder | 编码器理解输入，解码器生成输出 | T5、BART | 翻译、摘要 |

### Q3：什么是 LoRA？它为什么高效？（⭐⭐⭐⭐⭐）

**参考答案**：

LoRA（Low-Rank Adaptation）是一种参数高效微调方法。核心思想：
- 冻结预训练模型的原始权重矩阵 $W$。
- 引入两个小矩阵 $A$ 和 $B$（低秩分解），使得增量为 $\Delta W = BA$。
- 训练时只更新 $A$ 和 $B$，参数量从 $d \times d$ 降低到 $d \times r + r \times d$（$r \ll d$）。

**高效的原因**：
1. 可训练参数量减少 10-1000 倍。
2. 显存需求大幅降低。
3. 原始模型权重不变，可以快速切换不同的 LoRA 适配器。
4. 效果接近全参数微调。

### Q4：RLHF 的完整流程是什么？DPO 有什么改进？（⭐⭐⭐⭐⭐）

**参考答案**：

**RLHF 流程**：
1. **SFT 阶段**：用人工标注数据微调基础模型。
2. **奖励模型训练**：用人类偏好数据（对同一提示的多个回复排序）训练 Reward Model。
3. **PPO 优化**：用强化学习（PPO 算法）优化策略模型，使其生成的回复最大化奖励模型的评分。

**DPO 改进**：
- 直接用偏好数据优化策略模型，**跳过了奖励模型训练**。
- 将奖励函数隐式地嵌入到策略优化目标中。
- 训练更简单（不需要 PPO 的复杂采样），效果在某些场景下优于 RLHF。

### Q5：什么是幻觉（Hallucination）？分哪几类？如何系统性缓解？（⭐⭐⭐⭐⭐）

**参考答案**：

**幻觉分类**：
1. **事实性幻觉**：生成与客观事实不符的内容（如伪造论文引用）。
2. **忠实性幻觉**：生成内容与给定上下文不一致。
3. **输入幻觉**：对用户未提及的内容做出假设。

**系统性缓解方案**：
1. **RAG**：用检索到的真实文档作为生成基础。
2. **Prompt 设计**："如不确定请说不知道"、提供参考文档。
3. **Temperature 调低**：降低生成的随机性。
4. **RLHF/DPO**：通过人类偏好对齐减少幻觉。
5. **Self-Consistency**：采样多个回答取共识。
6. **引用标注**：要求模型标注信息出处。
7. **外部验证**：对生成内容进行事实核查。

### Q6：Temperature 参数如何影响 LLM 输出？实际中如何设置？（⭐⭐⭐）

**参考答案**：

Temperature 控制 softmax 分布的"平坦程度"：
- **T → 0**：分布尖锐，几乎总是选择最大概率的 Token（确定性）。
- **T = 1**：使用原始概率分布。
- **T > 1**：分布更平坦，增大低概率 Token 被选中的概率（更随机/创意）。

**实际设置建议**：
- 代码生成、数学推理：T = 0 ~ 0.2。
- 常规问答：T = 0.3 ~ 0.7。
- 创意写作、头脑风暴：T = 0.7 ~ 1.0。

### Q7：什么是上下文窗口（Context Window）？有什么限制？如何突破？（⭐⭐⭐⭐）

**参考答案**：

上下文窗口是 LLM 单次能处理的最大 Token 数量（如 GPT-4 Turbo 128K、Claude 3.5 200K）。

**限制**：超出窗口的信息会被截断或遗忘（"Lost in the middle" 问题）。

**突破方案**：
1. **RAG**：只检索相关的部分文档，不需要装载全文。
2. **长上下文模型**：使用原生支持长 context 的模型。
3. **滑动窗口**：分段处理文本。
4. **层次化摘要**：先摘要再处理。
5. **Sparse Attention**：只关注关键位置而非所有 Token。

### Q8：如何防止 PEFT 微调过程中的灾难性遗忘？（⭐⭐⭐）

**参考答案**：

1. LoRA 等 PEFT 方法本身天然缓解遗忘，因为原始权重被冻结。
2. 使用混合训练数据（新任务数据 + 原始能力数据）。
3. 正则化约束：限制新参数与原始参数的偏移。
4. 持续评估通用能力基准，确保不退化。

### Q9：RAG 和 Fine-tuning 的选择时机是什么？（⭐⭐⭐⭐⭐）

**参考答案**：

| 维度 | RAG | Fine-tuning |
|------|-----|-------------|
| 知识更新 | 实时更新，修改知识库即可 | 需重新训练 |
| 成本 | 推理时有检索开销，但无训练成本 | 训练成本高，推理成本低 |
| 适用数据 | 知识密集型、外部文档 | 行为/风格/特定任务类 |
| 幻觉控制 | 有文档兜底，更可控 | 记忆到模型中，可能过拟合 |
| 推荐策略 | 需要最新、可溯源的知识时使用 | 需要改变模型行为/输出风格时使用 |

**最佳实践**：很多生产系统同时采用 RAG + 轻量级微调。

### Q10：简述 Tokenization 中 BPE 算法的原理。（⭐⭐⭐）

**参考答案**：

BPE（Byte-Pair Encoding）：
1. 初始化词表为所有单字符。
2. 统计语料中所有相邻字符对的出现频次。
3. 将最高频的字符对合并为新的子词单元，加入词表。
4. 重复步骤 2-3 直到达到预设词表大小。

**优势**：在常见词和罕见词之间取得平衡，有效处理 OOV 问题。
