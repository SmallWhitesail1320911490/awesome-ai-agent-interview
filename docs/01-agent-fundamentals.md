# 第 1 章：Agent 核心概念与架构

> 本章系统梳理 AI Agent 的定义、核心组件、架构模式与推理循环，是 Agent 方向面试的重中之重。
> 内容基于 2024-2025 各大技术社区面经整理汇总（来源：牛客网、CSDN、小林coding、Medium、Analytics Vidhya 等）。

---

## 一、核心知识点总结

### 1.1 什么是 AI Agent

AI Agent（智能体）是一个能够**感知环境、自主决策并采取行动**以实现特定目标的系统，最小化人类干预。与传统 AI 模型不同，Agent 强调「自主性」「适应性」「交互性」。

现代 AI Agent 通常以大语言模型（LLM）为核心"大脑"，辅以记忆、工具调用、规划等模块。

### 1.2 Agent 的核心组件

根据 Lilian Weng 的经典博文及业界共识，Agent 包含以下核心组件：

| 组件 | 说明 | 典型实现 |
|------|------|----------|
| **大脑 (LLM)** | 负责理解、推理、规划和决策 | GPT-4o、Claude 3.5、Gemini 1.5 Pro |
| **记忆 (Memory)** | 短期记忆（上下文历史）+ 长期记忆（向量数据库） | Pinecone、Weaviate、ChromaDB |
| **规划 (Planning)** | 任务分解 + 自我反思 + 多步推理 | ReAct、CoT、ToT |
| **工具 (Tools)** | 与外界交互的能力 | Function Calling、MCP、API 调用 |
| **行动 (Action)** | 执行具体操作的能力 | 代码执行、搜索、数据库操作 |

### 1.3 Agent 架构设计模式

- **ReAct（Reason + Act）模式**：交替进行思考和行动，是最基础的 Agent 模式。
- **Tool-Use Agent**：调用外部 API 和工具完成任务。
- **Planner-Executor 模式**：先规划后执行，拆分规划 Agent 和执行 Agent。
- **Manager-Worker 模式**：管理 Agent 将任务委派给多个专业子 Agent。
- **Memory-Augmented Agent**：通过持久化记忆实现跨会话上下文保持。
- **CodeAct**：Agent 通过生成和执行代码来完成任务。
- **Agentic RAG**：将 RAG 与 Agent 流程结合，Agent 自主决定何时检索。

### 1.4 Agent 的推理循环

Agent 的核心运行机制是一个循环（Agent Loop）：

```
Observe（感知） → Think（思考/推理） → Act（行动） → Observe（观察结果） → ...
```

Agent 不断评估下一步行动、执行操作并考虑结果，直至任务完成或达到终止条件。

## 二、高频面试题

### Q1：什么是 AI Agent？它包含哪些核心组件？（⭐⭐⭐⭐⭐）

**参考答案**：

AI Agent 是一个能够自主感知环境、做出决策并采取行动以实现特定目标的智能系统。在基于 LLM 的上下文中，Agent 以大语言模型为"大脑"。

核心组件包括：
1. **大脑（LLM）**：负责理解、推理和决策的核心引擎。
2. **记忆（Memory）**：短期记忆保存当前对话上下文，长期记忆通过向量数据库实现跨会话信息持久化。
3. **规划（Planning）**：包含任务分解（Task Decomposition）和自我反思（Self-Reflection），使 Agent 能够处理复杂多步任务。
4. **工具（Tools）**：赋予 Agent 与外界交互的能力，如搜索网络、读写文件、执行代码、调用 API 等。
5. **行动（Action）**：将决策转化为具体操作的执行层。

### Q2：Agent 和纯大语言模型的主要区别是什么？（⭐⭐⭐⭐⭐）

**参考答案**：

| 维度 | 纯 LLM | AI Agent |
|------|--------|----------|
| 运行模式 | 被动应答，单轮输入输出 | 主动规划，多步自主执行 |
| 外界交互 | 无法访问外部系统 | 可通过工具调用感知和改变外部世界 |
| 记忆能力 | 仅依赖上下文窗口 | 拥有短期 + 长期记忆机制 |
| 知识新鲜度 | 受限于训练数据截止时间 | 可通过 RAG/搜索获取最新信息 |
| 任务复杂度 | 适合单步简单任务 | 适合复杂、开放式、多步骤任务 |

### Q3：请解释 ReAct 框架的核心思想和执行流程。（⭐⭐⭐⭐⭐）

**参考答案**：

ReAct（Reasoning and Acting）将推理和行动交替进行，核心流程为：
1. **Thought（思考）**：基于当前状态进行推理，决定下一步做什么。
2. **Action（行动）**：选择并调用合适的工具或执行操作。
3. **Observation（观察）**：获取工具执行的反馈结果。
4. 循环此过程直到获得最终答案（Finish）。

**优势**：
- 推理过程可解释、可追踪。
- 模型在行动前有逻辑推演，减少盲目操作。
- 解决复杂问题的成功率显著提升。

### Q4：请列举并比较常见的 Agent 架构设计模式。（⭐⭐⭐⭐）

**参考答案**：

| 模式 | 核心思想 | 适用场景 |
|------|----------|----------|
| ReAct | 推理与行动交替进行 | 通用任务，需要工具调用的场景 |
| Planner-Executor | 先生成完整计划，再逐步执行 | 复杂项目管理类任务 |
| Manager-Worker | 管理 Agent 分配任务给多个子 Agent | 需要多种专业能力协作的场景 |
| Reflexion | 执行后自我反思并改进 | 需要迭代优化的编程、写作任务 |
| CodeAct | Agent 通过写代码来实现任务 | 数据分析、自动化脚本等场景 |
| Agentic RAG | Agent 自主判断是否需要检索 | 知识密集型问答场景 |

### Q5：什么是认知架构（Cognitive Architecture）？在 Agent 设计中如何体现？（⭐⭐⭐）

**参考答案**：

认知架构是模拟人类认知过程的 AI 系统设计模式。在 Agent 中体现为：
- **感知模块**：理解用户输入和环境信息。
- **记忆模块**：类似人脑的短期工作记忆和长期经验记忆。
- **推理模块**：类似人的逻辑推理和决策。
- **学习模块**：从反馈中改进自身行为。
- **目标管理**：维护和更新任务目标。

### Q6：Agent 的"幻觉问题"如何产生？有哪些缓解策略？（⭐⭐⭐⭐）

**参考答案**：

Agent 的幻觉来源于底层 LLM 生成看似合理但与事实不符的内容。在 Agent 场景中尤其危险，因为可能导致错误的工具调用或决策。

缓解策略：
1. **RAG**：通过检索外部知识提供事实基础。
2. **Self-RAG**：让模型自主判断是否需要检索并自我验证。
3. **Prompt 约束**：明确指示"如不确定请回答不知道"。
4. **多 Agent 互审**：不同角色的 Agent 互相验证答案。
5. **置信度评分**：对生成结果进行置信度评估，低于阈值时拒绝回答或触发人工审核。
6. **Guardrails（护栏）**：设置输出安全边界和合规检查。

### Q7：什么时候应该使用 AI Agent，什么时候用传统的 Prompt Engineering 就够了？（⭐⭐⭐⭐）

**参考答案**：

**适合使用 Agent 的场景**：
- 需要多步骤推理和规划的复杂任务
- 需要与外部环境交互（搜索、数据库、API）
- 任务目标开放式，执行路径不确定
- 需要根据中间结果动态调整策略

**Prompt Engineering 即可的场景**：
- 单步且明确的任务（翻译、摘要、分类）
- 不需要外部工具调用
- 输出结构固定、可预测
- 延迟要求极高，无法容忍多轮推理开销

### Q8：在实际 Agent 开发中，有哪些常见的"坑"？（⭐⭐⭐⭐）

**参考答案**：

来自各大面经和实践总结：
1. **无限循环**：Agent 陷入反复调用同一工具的死循环，需设置最大迭代次数。
2. **上下文溢出**：多轮对话后超出 Token 限制，需做上下文压缩或摘要。
3. **错误传播**：前一步的错误结果被后续步骤继续使用，需加入验证机制。
4. **工具调用失败处理**：API 超时、格式错误等异常需要 fallback 策略。
5. **成本失控**：大量 Token 消耗导致 API 费用暴涨，需监控和限制。
6. **延迟过高**：多轮 LLM 调用串联导致端到端延迟不可接受，需优化并行度。

### Q9：如何评估一个 AI Agent 系统的性能？（⭐⭐⭐⭐）

**参考答案**：

评估维度包括：
1. **任务成功率**：最终是否正确完成目标任务。
2. **步骤效率**：完成任务所需的推理和工具调用步数。
3. **延迟**：端到端响应时间，包括首字延迟（TTFT）和总完成时间。
4. **成本**：Token 消耗和 API 调用费用。
5. **鲁棒性**：面对边界案例和异常输入时的表现。
6. **可解释性**：推理过程是否透明可追踪。
7. **安全性**：是否会生成有害内容或执行危险操作。

### Q10：设计一个生产级 Agent 系统时，要考虑哪些关键的工程问题？（⭐⭐⭐⭐⭐）

**参考答案**：

1. **可观测性（Observability）**：记录每一步推理和执行日志，便于调试和审计。
2. **安全护栏（Guardrails）**：Prompt 注入防护、输出安全审查、敏感操作需人工确认。
3. **失败恢复**：设计超时、重试、熔断和回滚机制。
4. **人机协作（Human-in-the-Loop）**：关键决策节点设置人工审核。
5. **成本控制**：Token 消耗监控、模型路由（小模型处理简单任务）、缓存策略。
6. **状态持久化**：支持长时间任务的中断和恢复。
7. **版本管理**：Prompt 模板和工具定义的版本控制。
