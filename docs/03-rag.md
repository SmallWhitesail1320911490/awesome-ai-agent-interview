# 第 3 章：RAG 检索增强生成

> RAG 是 Agent 面试中的绝对高频考点。本章涵盖完整 RAG 流水线、高级检索策略、前沿变体（GraphRAG / Self-RAG）。
> 内容来源：牛客网、CSDN、腾讯技术社区、阿里云开发者等。

---

## 一、核心知识点总结

### 1.1 RAG 标准流程

```
文档 → 文本提取 → 分块(Chunking) → Embedding → 向量数据库存储
                                                          ↓
用户 Query → Embedding → 向量检索(Top-K) → 重排序(Rerank) → 组装 Prompt → LLM 生成回答
```

### 1.2 关键技术点

- **Chunking 策略**：固定大小、按段落/句子、语义分块、结构化分块
- **Embedding 模型**：OpenAI text-embedding-3、BGE、E5、Sentence-BERT
- **向量数据库**：Milvus、Pinecone、Chroma、Weaviate、Qdrant、FAISS
- **混合检索**：BM25（稀疏）+ 向量（稠密）结合
- **重排序（Rerank）**：Cohere Rerank、BGE Reranker、Cross-Encoder
- **高级变体**：GraphRAG、Self-RAG、Agentic RAG、HyDE

## 二、高频面试题

### Q1：请简述 RAG 的完整流程。（⭐⭐⭐⭐⭐）

**参考答案**：

RAG 分为**离线索引**和**在线检索生成**两个阶段：

**离线索引阶段**：
1. 文档提取：从 PDF、Word、网页等提取纯文本。
2. 文本分块（Chunking）：按策略将长文档切分为小段。
3. 向量化：使用 Embedding 模型将每个 Chunk 转换为向量。
4. 存储：向量和原文存入向量数据库。

**在线检索生成阶段**：
1. 用户发起 Query。
2. 将 Query 向量化。
3. 在向量数据库中检索 Top-K 相似的 Chunk。
4. （可选）使用 Reranker 重排序，精选最相关的 Chunk。
5. 将检索结果与 Query 组装成 Prompt。
6. LLM 根据 Prompt 生成最终回答。

### Q2：RAG 中的 Chunking 策略如何选择？（⭐⭐⭐⭐⭐）

**参考答案**：

| 策略 | 原理 | 优势 | 劣势 |
|------|------|------|------|
| 固定大小分块 | 按 Token 数切分（如 512 Token） | 简单高效 | 可能在句子中间切断 |
| 按段落/句子 | 以自然段或句号分界 | 保持语义完整 | 长度不均匀 |
| 语义分块 | 用 Embedding 相似度检测语义边界 | 更精准 | 计算开销大 |
| 递归分块 | 逐层尝试不同分隔符 | LangChain默认推荐 | 需要调参 |
| 结构化分块 | 按 Markdown 标题、HTML 标签切分 | 保留文档结构 | 依赖文档格式 |

**一般建议**：Chunk 大小 256-1024 Token，设置 10-20% Overlap（重叠），避免信息在边界丢失。

### Q3：混合检索是什么？为什么比纯向量检索效果好？（⭐⭐⭐⭐⭐）

**参考答案**：

混合检索结合**稀疏检索**（BM25/TF-IDF）和**稠密检索**（向量相似度）：
- **BM25 擅长**：精确关键词匹配、专有名词、数字查询。
- **向量检索擅长**：语义理解、同义词匹配、模糊查询。

通过加权融合两者的分数（如 RRF 倒数排序融合），可以兼顾精确匹配和语义理解，召回率和精确率双提升。

### Q4：什么是 GraphRAG？它解决了传统 RAG 的哪些问题？（⭐⭐⭐⭐⭐）

**参考答案**：

GraphRAG 利用知识图谱（Knowledge Graph）增强 RAG 的检索和推理能力。

**传统 RAG 的痛点**：
- 只能做"单跳"检索，难以回答需要跨多个文档关联推理的问题。
- 无法理解实体间的复杂关系。

**GraphRAG 的核心创新**：
1. 用 LLM 从文档中自动抽取实体和关系，构建知识图谱。
2. 对知识图谱做层次化社区发现（Community Detection）。
3. 检索时不仅检索文本块，还检索实体关系路径。
4. 支持**多跳推理**：从 A → B → C 的关系链推理。

**应用场景**：复杂的企业知识库问答、需要跨文档关联推理的场景。

### Q5：什么是 Self-RAG？核心创新点是什么？（⭐⭐⭐⭐）

**参考答案**：

Self-RAG（Self-Reflective RAG）让模型**自主决定何时检索、是否使用检索结果**。

核心创新：
1. 引入特殊的**反思 Token**（如 `[Retrieval]`、`[No Retrieval]`、`[IsREL]`、`[IsSUP]`）。
2. 模型在生成过程中动态判断：是否需要检索？检索结果是否相关？生成结果是否被检索结果支持？
3. 不需要额外的检索决策模块，**端到端训练**。

**与标准 RAG 的对比**：
- 标准 RAG：每次都检索 → 可能引入噪声。
- Self-RAG：按需检索 → 减少不必要的检索开销，提升事实准确性。

### Q6：如何评估 RAG 系统的效果？（⭐⭐⭐⭐）

**参考答案**：

分**检索阶段**和**生成阶段**评估：

**检索阶段**：
- Precision@K / Recall@K：Top-K 中有多少是真正相关的。
- MRR（Mean Reciprocal Rank）：第一个正确结果的排名倒数。
- NDCG：归一化折损累积增益。

**生成阶段**：
- 事实准确性：生成内容是否与检索文档一致（Faithfulness）。
- 答案相关性：回答是否命中用户问题。
- BLEU / ROUGE：与参考答案的文本重合度（定量）。
- 幻觉检测：是否生成了检索文档中没有的错误信息。

**端到端评估框架**：RAGAS、TruLens。

### Q7：如何处理 RAG 中"检索到了但答案不对"的问题？（⭐⭐⭐⭐）

**参考答案**：

这是"忠实性不足"问题，常见解决方案：
1. **Prompt 约束**：明确"仅根据以下文档内容回答，不要加入额外知识"。
2. **Reranker 精排**：用 Cross-Encoder 精确判断 Chunk 与 Query 的相关度。
3. **引用标注**：要求模型标注回答中每句话对应的来源段落。
4. **置信度评分**：对回答进行置信度评估，低置信度触发人工介入。
5. **Self-RAG**：模型自主评估检索结果是否支持当前生成。

### Q8：RAG 如何处理多文档之间的信息冲突？（⭐⭐⭐⭐）

**参考答案**：

1. 让 LLM 总结共识点，并**明确标注分歧**（通过 Prompt 控制）。
2. 按文档**来源权威性加权**（如官网 > 博客 > 论坛）。
3. 按**时间戳排序**，优先引用最新的信息。
4. 返回**多视角答案**，让用户自行判断。
5. 在 Prompt 中明确告知"检索到的文档存在矛盾，请综合分析"。

### Q9：如何优化长文档的检索效果？（⭐⭐⭐⭐）

**参考答案**：

1. **Small-to-Big**：先检索小 Chunk，命中后关联其所属的大段落或整个章节。
2. **层次检索**：先定位章节级别，再在章节内做细粒度 Chunk 检索。
3. **Parent Document Retrieval**：存储 Chunk 时保留其父文档引用。
4. **图结构检索**：利用知识图谱关联文档片段。
5. **递归摘要**：对长文档逐层摘要，检索时先匹配摘要再深入原文。

### Q10：什么是 HyDE（Hypothetical Document Embeddings）？（⭐⭐⭐）

**参考答案**：

HyDE 的核心思路：不直接用 Query 检索，而是先让 LLM 生成一段"假设性的理想回答"，然后用这段假设文档的 Embedding 去检索。

**优势**：假设文档比原始 Query 更接近目标文档的语义表达，提升检索相关性。
**局限**：增加了一次 LLM 调用的延迟和成本；假设文档质量依赖 LLM 能力。

### Q11：什么是 Agentic RAG？（⭐⭐⭐⭐）

**参考答案**：

Agentic RAG 将 RAG 与 Agent 流程结合：
- Agent 自主决定**是否需要检索**（而非每次都检索）。
- Agent 可以**多次迭代检索**，根据第一次检索结果决定是否需要更多信息。
- Agent 可以选择**不同的检索工具**（如向量检索、关键词检索、数据库查询）。
- Agent 可以对检索结果进行**验证和过滤**后再生成。

这使得 RAG 从"被动检索"升级为"主动智能检索"。
