# 贡献指南

感谢您考虑为 **Awesome AI Agent Interview** 做出贡献！这个项目依赖于社区的力量来保持内容的准确性和及时性。

## 我们欢迎什么样的贡献？

- 🚀 **补充新的面试题**：如果您在面试中遇到了本项目未收录的 Agent 相关问题。
- 📖 **完善现有答案**：如果您发现某个答案不够严谨、清晰，或者有更好的补充。
- 🐛 **修复错别字或失效链接**：帮助我们保持文档的整洁和专业。
- 💡 **提供新的学习资源**：推荐优秀的博客、论文或视频教程。

## 如何参与贡献

### 详细步骤

1. **Fork 本仓库**：点击仓库页面右上角的 Fork 按钮。
2. **克隆您的仓库**：
   ```bash
   git clone https://github.com/您的用户名/awesome-ai-agent-interview.git
   ```
3. **创建分支**（建议以您要修改的内容命名）：
   ```bash
   git checkout -b feature/add-react-questions
   ```
4. **进行修改**：请确保您的修改遵循以下的 Markdown 规范。
5. **提交代码**：
   ```bash
   git add .
   git commit -m "feat: 新增关于 ReAct 框架的 3 道面试题"
   ```
6. **推送到您的仓库**：
   ```bash
   git push origin feature/add-react-questions
   ```
7. **提交 Pull Request (PR)**：回到本仓库页面，点击 `New Pull Request`。

## 内容规范

为了保持项目的整体一致性，请在修改时遵循以下规范：

- 格式：使用标准的 Markdown 语法。
- 语言：目前本项目主要以中文编写。
- 结构：每道面试题请采用一致的格式（比如加粗题目，使用无序列表列出答案要点）。
- 引用：如果答案来自他人的文章或官方文档，请在答案末尾添加参考链接。

## Commit 信息规范

我们建议使用以下的前缀：
- `feat`: 新增内容（面试题、章节等）
- `fix`: 修改错误或完善答案
- `docs`: 文档修改（README 或指南）
- `style`: 格式调整（不影响内容的变动，如空格、缩进）
- `chore`: 其他无需修改内容的变动

感谢您的参与，让我们一起打造最优质的 AI Agent 面试指南！
